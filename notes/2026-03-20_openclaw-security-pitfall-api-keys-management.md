# OpenClaw安全踩坑：API密钥别裸奔

## 内容
### 关键要点
- 核心告警：明文API key暴露是高频、致命、低级错误。
- 风险场景：配置误传、公开仓库泄露、控制面板暴露、公网调试。
- 后果：账单爆炸、账号后门、聊天记录/云账号/钱包被接管。
- 防坑思路：先收敛密钥管理，再扩展功能。
- 结论：OpenClaw能力强，但错误安全姿势会反噬。

### 方法步骤（如有）
1. 全文检索并删除明文key。
2. 改为环境变量注入。
3. 子账号+预算上限+告警+定期轮换。
4. Control UI仅本地绑定或加严鉴权。

- “API密钥别裸奔。”
- “先把key藏好，再玩agent花活。”
