# 2026 最火岗位：AI Engineer 入场指南

## 内容

有一个方向这两年悄悄在放水，就是 AI Engineer，只要你混进去了，真的很容易做出 impact。AI Engineer 是 LLM 出来之后的新岗位，做 RAG、AI Agent、文档问答这些。底层是 Search + Distributed system + Infra，不需要懂 ML 算法。门槛没那么高，但岗位是真的多。

### 一个 AI Engineer 需要会什么？三块：

**1️⃣ AI 基础**
LLM 怎么用、prompt tuning、RAG pipeline、embedding、agent 架构。不用推到 attention 公式。

**2️⃣ 工程化**
把模型能力包成 API，能 deploy、debug，懂一点 CI/CD 和 system stability。

**3️⃣ 系统整合**
接 PostgreSQL、向量库、外部工具、业务接口。有点成本意识和产品思维。

不需要全都会，但面试前至少把能帮你进场的那部分拿下。现在很多厂的 AI 岗，面试重点已经不是手推反向传播，而是你怎么搭出一个能跑的 end-to-end AI pipeline。

所以别只刷题做传统 CRUD 了。AI + Full Stack 的 combo，简历上亮眼太多。一个普通的 To-do App 面试官看都不想看，但一个完整的 AI 驱动的全栈项目，能直接证明你跟得上行业节奏。

### 推荐实战项目：DocuAI

- **前端**：Next.js，展示对 React SSR 的理解
- **后端**：FastAPI 或 Express，接 OpenAI API 或 LangChain，做文档智能解析 + 问答
- **数据层**：PostgreSQL + 向量数据库，存文档 embedding，实现语义检索

这一套做下来，从 UI、API、AI 集成到数据层，技能树清清楚楚。简历上放这个，比纯刷出来的项目硬核太多。与其在传统 SDE 方向死磕，不如花几周做个 AI + Full Stack 项目，更容易突出重围。
