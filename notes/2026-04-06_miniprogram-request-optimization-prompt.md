# Prompt：小程序页面请求逻辑优化

## 内容

**Prompt for bug fixes for app:**

请优化这个小程序页面的请求逻辑。

要求：
1. 不要只修功能，要检查重复请求。
2. 统计 onLoad、onShow、组件初始化、watcher 中所有接口调用。
3. 同一个接口只能有一个统一请求入口。
4. 增加 loading 防重复提交。
5. 增加 hasLoaded / needRefresh 状态，避免页面切换时重复拉取。
6. 对适合缓存的数据设计缓存策略。
7. 输出优化前后的请求次数对比。
8. 不要改变现有业务功能。

## Bubble 的洞察

- 这个 prompt 的价值在第7条：要求输出"优化前后请求次数对比"——让 AI 自己量化改动效果，这是把 AI 从执行者升级为审计者的关键一步。可以复用到任何需要性能优化的 AI 对话里。
